# Open Questions

Status: Draft  
Date: 2026-06-13

These questions are deliberately unresolved. They should not block V1 unless marked as launch-critical.

## V1 Launch-Critical

- What are the first 5-10 topics?
- What is the minimum real content count required for launch?
- Which newsletter or subscription provider should be used?
- What privacy-aware analytics approach, if any, is acceptable?
- Which share surfaces should ship first beyond native share and copy link?
- Should tool announcement articles use the same template as essays at launch?
- Should CMS users be linkable to public people records, or should public people remain separate editorial records?
- What open-source license should govern the codebase?
- Should content use the same license as the codebase or a separate license?
- What final name and domain should replace the TEMIS codename, if any?

## V1 Design

- Should the visual system start light-first, dark-first, or support both from launch?
- How much motion should the homepage use?
- How prominent should the topic relationship navigation be on the homepage?
- What is the mobile interaction model for topic relationship navigation?
- Should author pages prioritize biography, authored articles, external links, or topic expertise?
- What default visual language should placeholder content use before final art direction exists?

## V1 Content And Editorial

- What are the editorial quality rules for essays?
- What are the editorial quality rules for tool announcement articles?
- Should all articles have named people, publication-owned bylines, or both?
- What author/person profile fields are required at launch?
- Should topics support editorial descriptions long enough to become landing pages?
- How should related articles be selected before there is enough content for automation?

## V1 Technical

- What URL structure should articles, topics, and people use?
- What SEO defaults should be generated for records with missing metadata?
- What share image defaults should be generated for records with missing images?
- What media sizes and aspect ratios should editors be encouraged to use?
- What acceptance criteria should CMS preview satisfy before launch?
- How should build failures after publish be surfaced to editors?
- How should local lorem ipsum content be replaced by CMS-managed content during launch hardening?

## Later Questions

These are intentionally not part of V1 scope.

- Should topics support hierarchy later?
- Should future request-like records separate type from instance?
- Should relationships require evidence or editorial rationale?
- Should public records have health/status metadata independent of publication status?
- Should relationship confidence or weighting ever exist?
- How should duplicate detection work?
- How should record merging work?
- How are future curators appointed or removed?
- How are appeals handled if public contribution intake is added?
- How are conflicts of interest declared if public contribution intake is added?
- Should any form of public voting ever exist?
- Should automated grouping or recommendation features ever exist?
- Should future matching, opportunity discovery, or gap-detection features exist?
